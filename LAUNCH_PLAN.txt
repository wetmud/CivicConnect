PHASE 1: MUST DO BEFORE SHARING (blockers)
═══════════════════════════════════════════════════════════════

1. REPLACE GEOAPIFY KEY EXPOSURE
   Status: Hardcoded at line ~2061 (def29eda434347f6b7f2e0cdcbc47ce8)
   Risk: Anyone can view-source, steal the key, burn your quota
   Fix: Cloudflare Worker proxy (free tier, ~30 min setup)
   - Create Worker that proxies requests to api.geoapify.com
   - Worker holds the key server-side
   - Frontend calls your Worker URL instead
   - Add rate limiting in the Worker (optional but smart)

2. REPLACE CORSPROXY.IO
   Status: Hardcoded at line ~2062, used for OpenParliament API
   Risk: Third-party proxy you don't control — could go down, intercept data
   Fix: Same Cloudflare Worker can proxy OpenParliament requests too
   - One Worker, two routes: /geoapify/* and /openparliament/*
   - Or use Vercel Edge Function if you prefer
   - Update CSP connect-src to your worker domain

Both of these can be ONE Cloudflare Worker. Estimated effort: 1-2 hours.

WHAT NOT TO DO
═══════════════════════════════════════════════════════════════

- Don't wait for the meeting scraper to work. It's commented out. Ship without it.
- Don't wait for the budget tab. It's commented out. Ship without it.
- Don't add new features before launching. You have enough.
- Don't set up analytics before launching. Add it when you care about numbers.
- Don't buy a domain before launching. GitHub Pages URL works fine for v1.
- Don't refactor to modules/framework. Single file is fine at this scale.

The app works. The core value prop (address → reps → email) is solid.
Ship it, share it, learn from real users.
